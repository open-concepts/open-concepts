بند recipe books کا زمانہ

Linux سے پہلے، computing ایسی تھی جیسے کوئی restaurant ہو جہاں recipe ایک بند تجوری میں رکھی ہو۔ آپ کھانا کھا سکتے تھے، لیکن یہ نہیں دیکھ سکتے تھے کہ کیسے بنا یا recipe بدل سکتے تھے۔ ۱۹۴۰ اور ۱۹۵۰ کی دہائی کے computers میں کوئی operating system نہیں تھا — program ایک ایک کرکے physical switches سے load ہوتے تھے۔

سن ۱۹۹۱ میں Finland کے ایک طالب علم Linus Torvalds نے ایک مفت اور کھلا متبادل بنایا — Linux kernel۔ انہوں نے سب کے لیے recipe کھول دی۔

اخروٹ کی مثال

Operating system ایک اخروٹ کی طرح ہے۔ بالکل درمیان میں ہے kernel — سخت خول کے اندر کی گری۔ اس کے گرد ہے shell — سخت باہری تہہ۔ Shell آپ کی باتوں کو ایسی زبان میں بدلتی ہے جو گری سمجھ سکے۔ آپ، user، باہر بیٹھے پورا اخروٹ تھامے shell سے بات کر رہے ہیں۔

Kernel آپ سے کبھی براہ راست بات نہیں کرتا۔ وہ صرف hardware سے بات کرتا ہے۔ Bash وہ shell ہے — ترجمان جو آپ کے plain commands کو وہ زبان میں بدلتا ہے جو Kernel سمجھتا ہے۔

🧠 Kernel کے چار اہم کام

🧠 Memory Management

Kernel فیصلہ کرتا ہے کہ کون سا program computer کی RAM (وہ جگہ جہاں چل رہی چیزیں رہتی ہیں) استعمال کرے گا۔ یہ یقینی بناتا ہے کہ programs ایک دوسرے کی memory نہ چرائیں — جیسے Master Chef فیصلہ کرے کہ کون سی dish کو سب سے بڑا دیگ ملے۔

⚙️ Process Management

Kernel یقینی بناتا ہے کہ ہر program کو CPU استعمال کرنے کا موقع ملے۔ جیسے ایک chef ایک ساتھ کئی dishes سنبھالے — نہ کچھ جلے، نہ کچھ رہ جائے۔

🔌 Device Drivers

Kernel hardware کے لیے universal ترجمان کا کام کرتا ہے۔ وہ آپ کی screen، keyboard، printer سب سے بات کرنا جانتا ہے — تاکہ آپ کے programs کو ہر brand کی الگ زبان نہ سیکھنی پڑے۔

🛡️ System Security

Kernel ایک چوکیدار کی طرح ہے — یہ یقینی بناتا ہے کہ ایک user دوسرے کی private files نہ دیکھ سکے اور نہ computer کو نقصان پہنچا سکے۔ ہر request کو execute کرنے سے پہلے permission rules چیک کی جاتی ہیں۔

اہم بات

Shell کی ضرورت کیوں ہے؟

Kernel بہت طاقتور ہے، لیکن وہ انسانی زبان نہیں سمجھتا۔ وہ صرف پیچیدہ electrical signals اور binary numbers جانتا ہے۔ Bash (یعنی Bourne Again Shell) دنیا کا سب سے مشہور shell ہے — ایک waiter جو آپ کا order plain زبان میں (ls -la) لے کر Master Chef کی زبان میں بدل دیتا ہے۔

جب آپ command لکھتے ہیں تو کیا ہوتا ہے
# آپ لکھتے ہیں (انسانی زبان)
$ ls -la /home/alice

# Bash (waiter) اسے translate کر کے kernel کو بھیجتا ہے
# Kernel file system پڑھتا ہے → data واپس کرتا ہے
# Bash نتیجہ format کر کے آپ کی screen پر دکھاتا ہے

drwxr-xr-x alice alice 4096 Jan 01 Documents
-rw-r--r-- alice alice 512 Jan 01 notes.txt

ایک بڑا گھر جس کے کمروں کے نام ہیں

Windows میں drives کو حروف ملتے ہیں جیسے C: یا D:، لیکن Linux میں سب کچھ ایک ڈھانچے میں ہوتا ہے جسے root کہتے ہیں — یعنی ایک آگے کی slash /۔ ہر file، program، حتیٰ کہ hardware بھی اسی ایک درخت کی شاخ ہے جو / سے شروع ہوتی ہے۔

Computer کو ایک بڑا گھر سمجھیں۔ File system وہ طریقہ ہے جس سے کمرے اور الماریاں ترتیب دی گئی ہیں۔ ہر کمرے کا ایک مقصد ہے، اور ہر چیز کا ایک path ہے — ان کمروں کی فہرست جن سے گزر کر آپ وہاں پہنچتے ہیں۔

🏠 Linux گھر کے اہم کمرے

/ — سب کا آغاز
/ ← پورے گھر کا مرکزی دروازہ
├── bin/ ← ضروری اوزاروں کی دراز (ls, cp, mv...)
├── sbin/ ← تہہ خانہ — مالک کے بھاری اوزار
├── etc/ ← قاعدوں کی کتاب — ہر program کی settings
├── home/ ← بیڈروم — ہر عام user کا الگ فولڈر
│ └── alice/ ← Alice کا ذاتی کمرہ
├── root/ ← مالک کا private suite (صرف superuser)
├── var/ ← گھر کی log book (logs، mail queues)
├── tmp/ ← ردی کاغذ (reboot پر صاف ہو جاتا ہے)
├── dev/ ← Hardware کی الماری (keyboard، mouse بھی files ہیں)
├── proc/ ← گھر کے sensor — live kernel data (virtual)
└── sys/ ← مزید sensor — hardware کی حالت (virtual)
/bin

ضروری اوزاروں کی دراز

بنیادی executable tools جن کی ہر user کو ضرورت ہے۔ commands جیسے ls, cp, mv, cat۔ اگر یہ خراب ہوں تو سارا گھر بیکار ہو جائے۔

/sbin

تہہ خانہ

بڑی مرمت کے system binaries۔ صرف root (مالک) user کو یہاں آنا چاہیے — جیسے boiler یا fuse box۔ عام users کا یہاں کوئی کام نہیں۔

/etc

قاعدوں کی کتاب

ہر installed program کی configuration files۔ جیسے دیوار پر لگا manual جو chefs کو بتاتا ہے کہ oven کا درجہ حرارت کیا رکھنا ہے۔ یہاں ایک file بدلو اور پورے program کا رویہ بدل جائے گا۔

/home

بیڈروم

ہر عام user کا ذاتی فولڈر۔ /home/alice, /home/bob۔ صرف اسی کمرے کا مالک اندر جا سکتا ہے۔ آپ کی Documents، Pictures اور config files یہیں رہتی ہیں۔

/root

مالک کا Private Suite

Superuser کا home folder — عام بیڈرومز سے بالکل الگ رکھا گیا ہے۔ System کا سب سے طاقتور account یہاں رہتا ہے۔

/var

گھر کی Log Book

بدلتی رہنے والی files: system logs، mail queues، databases۔ جیسے وہ log book جو ریکارڈ رکھے کہ ڈاک کب آئی اور کون سا bulb جلا۔

/tmp

ردی کاغذ کا ڈھیر

وہ temporary files جو programs کام کرتے وقت بناتے ہیں۔ System ہر reboot پر یہ جگہ صاف کر دیتا ہے — جیسے حساب کر کے notepad پھینک دینا۔

/dev

Hardware کی الماری

Linux میں ہر hardware device ایک file ہے۔ آپ کا keyboard ہے /dev/input/keyboard۔ Hard drive ہے /dev/sda۔ سب کچھ ایک file ہے — Linux کا بنیادی اصول۔

/proc & /sys

گھر کے Sensor — Virtual کمرے

یہ directories hard drive پر physically موجود نہیں ہیں۔ انہیں kernel live بناتا ہے اور یہ real-time معلومات دیتی ہیں — CPU کا استعمال، memory، چل رہے processes۔ جیسے دیوار پر لگے sensor جو ابھی کے درجہ حرارت اور نمی دکھائیں۔

آسان وضاحت

گھر میں آنکھیں کھولنا

ہر Bash command تین حصوں پر چلتی ہے: Command Name (کیا کرنا ہے)، Options/Flags (کیسے کرنا ہے — dash کے ساتھ)، اور Argument (کس پر کرنا ہے)۔ یہ تین navigation commands سیکھ لیں اور Linux کے file system میں کچھ بھی تلاش کر سکتے ہیں۔

👁️ ls — آنکھیں کھولیں

ls command آپ کو بتاتی ہے کہ کسی folder کے اندر کیا ہے۔ جیسے آنکھیں کھول کر کمرے میں دیکھنا۔ یہ وہ پہلی command ہے جو ہر Linux user سیکھتا ہے۔

Optionنامکیا کرتی ہے
-aسبچھپی ہوئی چیزیں بھی دکھائے (dot سے شروع ہونے والی files) — جیسے counter کے نیچے کی خفیہ دراز
-lتفصیلپوری تفصیل دکھائے: permissions، مالک، size، اور آخری بار بدلنے کا وقت
-hسمجھنے کے قابلbytes کو پڑھنے کے قابل بنائے: KB، MB، GB
-RRecursiveہر folder کے اندر جھانکے، اور ان کے اندر بھی — مکمل تلاش
-tوقتچیزوں کو آخری بار بدلنے کے وقت کے حساب سے ترتیب دے — نیا پہلے
-SSizeچیزوں کو file size کے حساب سے ترتیب دے — سب سے بڑی پہلے
-rالٹاکسی بھی ترتیب کو الٹا کر دے
ls examples
# بنیادی — اس folder میں کیا ہے؟
$ ls
Documents Downloads notes.txt script.sh

# Professional combo: تفصیل + سمجھنے والا + چھپی files
$ ls -lah
drwxr-xr-x alice alice 4.0K Jan 10 Documents
-rw-r--r-- alice alice 512 Jan 10 notes.txt
-rw------- alice alice 256 Jan 09 .bash_history ← چھپی ہوئی

🚶 cd — دروازوں سے گزرنا

cd کے ضروری shortcuts

  • cd DocumentsDocuments کمرے میں جائیں
  • cd ..ایک level اوپر جائیں — parent کمرہ
  • cd ~فوری طور پر اپنے home bedroom میں جائیں
  • cd -پچھلے کمرے میں واپس جائیں (back button)
  • cd /مرکزی دروازے تک پہنچیں — سب کی جڑ

Absolute بمقابلہ Relative Paths

# Absolute — / سے شروع ہوتا ہے
$ cd /home/alice/Documents

# Relative — جہاں آپ ابھی ہیں وہاں سے
$ cd Documents/Projects

📍 pwd — میں کہاں ہوں؟

جب آپ nested folders میں کہیں کھو جائیں، pwd آپ کو بتائے گا کہ آپ کہاں ہیں — مرکزی دروازے / سے آپ کے موجودہ کمرے تک پورا راستہ۔ Scripts چلاتے وقت خاص طور پر ضروری ہے۔

pwd
$ pwd
/home/alice/Documents/Projects/website

# آپ ابھی مرکزی دروازے سے پانچ کمرے اندر ہیں

بنانا، منتقل کرنا، اور مٹانا

Navigation سے آپ دیکھ سکتے ہیں۔ Action commands سے آپ چیزیں بدل سکتے ہیں — نئے کمرے بنائیں، چیزوں کی copies بنائیں، ایک جگہ سے دوسری جگہ منتقل کریں، اور مٹائیں۔ سب سے اہم اصول: Linux server پر کوئی Trash Can نہیں ہوتا۔ مٹا دیا مطلب ہمیشہ کے لیے گیا۔

🏗️ mkdir — نیا کمرہ بنائیں

ایک نیا folder بناتا ہے۔ -p flag ایک ساتھ پوری nested rooms کی gallery بنا دیتا ہے — ایک ایک کر کے بنانے کی ضرورت نہیں۔

$ mkdir Reports ← ایک کمرہ
$ mkdir -p Projects/2026/January ← ایک ساتھ پوری gallery

📄 touch — خالی file بنائیں

ایک نئی، خالی file بناتا ہے — جیسے میز پر خالی کاغذ رکھنا۔ File کا مواد بدلے بغیر اس کا "آخری تبدیلی کا وقت" بھی update کر سکتا ہے۔

$ touch notes.txt ← نئی خالی file
$ touch notes.txt ← timestamp update کرے (file وہی رہے)

📋 cp — Copy کریں

ایک نقل بناتا ہے — جیسے photo کو copy machine میں ڈالنا۔ اصل اپنی جگہ رہتی ہے؛ نئی copy مقصد کی جگہ ظاہر ہوتی ہے۔

Optionناماستعمال
-rRecursiveپورا folder اور اس کی تمام چیزیں copy کرے — directories کے لیے ضروری
-pPreserveاصل مالک، timestamps اور permissions copy میں بھی رکھے
-uUpdateصرف تب copy کرے جب source زیادہ نئی ہو — smart sync
$ cp notes.txt backup/notes.txt ← ایک file
$ cp -rp Projects/ /backup/Projects/ ← پورا folder، محفوظ

🚚 mv — منتقل کریں اور نام بدلیں

Linux میں file کو منتقل کرنا اور نام بدلنا ایک ہی command ہے۔ آپ بس ایک چیز اٹھا کر کہیں اور رکھ رہے ہیں یا نیا نام دے رہے ہیں۔ اصل اپنی پرانی جگہ سے ختم ہو جاتی ہے۔

$ mv notes.txt Documents/notes.txt ← دوسرے کمرے میں لے جائیں
$ mv old_name.txt new_name.txt ← اسی جگہ نام بدلیں

🗑️ rm — مٹائیں ⚠️ واپس نہیں آتا

Files ہمیشہ کے لیے مٹا دیتا ہے۔ Server پر کوئی Trash Can نہیں، کوئی Recycle Bin نہیں، کوئی Undo نہیں۔ مٹ گیا تو مٹ گیا۔ یہ وہ command ہے جو نئے users کے لیے سب سے خطرناک ہے۔

Optionنامکیوں استعمال ہوتی ہے
-iInteractive✓ نئے users کے لیے بہتر — ہر بار مٹانے سے پہلے پوچھے "یقین ہے؟"
-rRecursiveپورا folder اور اس کے اندر کی ہر چیز مٹائے
-fForce⚠️ بغیر پوچھے مٹائے، protected files بھی۔ انتہائی احتیاط سے استعمال کریں
$ rm -i notes.txt ← محفوظ: مٹانے سے پہلے پوچھتا ہے
rm: remove regular file 'notes.txt'? y

$ rm -rf Projects/ ← خطرناک: پورا folder خاموشی سے مٹا دیتا ہے
⚠️ کوئی output نہیں۔ کوئی تصدیق نہیں۔ بس گیا۔

Linux servers بہت زیادہ text پیدا کرتے ہیں: log files، config files، output data۔ آپ کو ایسے tools چاہئیں جو انہیں پڑھ سکیں اور ہزاروں lines میں سے ٹھیک وہ line نکال سکیں جو آپ ڈھونڈ رہے ہیں۔ یہ تین commands آپ کی عینک اور magnifying glass ہیں۔

📖 cat — پوری کتاب پڑھیں

پوری file کا مواد screen پر دکھاتا ہے۔ جیسے کتاب کھول کر ہر صفحہ ایک ساتھ پڑھنا۔ چھوٹی files کے لیے بہتر ہے — بڑی files بہت تیزی سے گزر جاتی ہیں۔

$ cat notes.txt ← پوری file دکھائیں
$ cat -n notes.txt ← line نمبروں کے ساتھ
$ cat file1.txt file2.txt ← دو files جوڑ کر دکھائیں

🔎 grep — Magnifying Glass

کسی file یا stream میں کوئی خاص لفظ یا pattern تلاش کرتا ہے۔ جیسے ایک بڑی phone book میں magnifying glass سے کوئی نام ڈھونڈنا۔ Linux کی سب سے طاقتور اور سب سے زیادہ استعمال ہونے والی commands میں سے ایک۔

-i

Case-Insensitive

"Error"، "ERROR"، "error" سب ملائے

-v

الٹا Match

لفظ کے علاوہ باقی سب دکھائے

-r

Recursive

ہر folder کی ہر file میں تلاش کرے

# log file میں "error" تلاش کریں (چھوٹے بڑے حروف نظر انداز)
$ grep -i "error" /var/log/syslog

# logs folder کی ہر file میں تلاش
$ grep -r "Failed login" /var/log/

# وہ تمام lines دکھائیں جن میں "debug" نہیں
$ grep -v "debug" app.log

📰 head & tail — شروع یا آخر دیکھیں

head file کی پہلی ۱۰ lines دکھاتا ہے۔ tail آخری ۱۰ دکھاتا ہے۔ دونوں -n سے تعداد بدلی جا سکتی ہے۔ tail -f professionals کا live monitoring tool ہے — یہ file کو دیکھتا رہتا ہے اور نئی lines آتے ہی دکھاتا ہے۔

$ head -n 5 server.log ← پہلی 5 lines
$ tail -n 20 server.log ← آخری 20 lines

# Professional live-monitoring trick:
$ tail -f /var/log/syslog ← live news ticker
Jan 10 14:32:01 server sshd: Accepted login for alice
Jan 10 14:32:15 server cron: Running daily backup...

آسان وضاحت

تین چابیاں اور تین گروہ

ایک ایسے گھر میں جہاں کئی لوگ رہتے ہوں، ہر file کے تین قسم کی چابیاں ہیں — Read (پڑھنا)، Write (لکھنا)، اور Execute (چلانا) — اور تین گروہ جن کے پاس یہ چابیاں ہو سکتی ہیں: Owner (آپ)، Group (آپ کی team)، اور Others (باقی سب)۔ یہ combination طے کرتا ہے کہ کون کیا کر سکتا ہے۔

🗝️ تین چابیاں

👁️

r = 4

Read — پڑھنا

file دیکھنے یا folder کا مواد جاننے کی اجازت۔ یہ چابی نہیں تو file آپ کو نظر ہی نہیں آئے گی۔

✏️

w = 2

Write — لکھنا

file بدلنے، مٹانے یا folder میں نئی چیزیں شامل کرنے کی اجازت۔ یہ چابی حفاظت سے دیں — سوچ کر کسی کو دیں۔

▶️

x = 1

Execute — چلانا

file کو program کی طرح چلانے یا folder میں cd سے داخل ہونے کی اجازت۔ یہ چابی نہیں تو script وہیں بیکار پڑی رہے گی۔

🏷️ Permission Label پڑھنا

جب آپ ls -l چلاتے ہیں تو ایک string نظر آتی ہے جیسے -rwxr-xr--۔ یہ Lock Label ہے، چار حصوں میں:

-rwx r-x r-- alice staff 4096 Jan 10 script.sh

│ └─────┘ └─────┘ └─────┘
│ OWNER GROUP OTHERS
└── file type (- = file, d = directory)

🔢 نمبروں کا نظام

نمبرعلامتچابیاںمطلب
7rwx4+2+1پڑھ، لکھ، اور چلا سکتا ہے — مکمل اختیار
6rw-4+2پڑھ اور بدل سکتا ہے، لیکن program نہیں چلا سکتا
5r-x4+1پڑھ اور چلا سکتا ہے، لیکن بدل نہیں سکتا
4r--4صرف دیکھ سکتا ہے — read-only
0---0کچھ نہیں کر سکتا — مکمل بند
chmod examples
# owner کو سب چابیاں (7)، باقیوں کو پڑھنے+چلانے کی (5)
$ chmod 755 script.sh

# ذاتی file: صرف owner پڑھ/لکھ سکے (6)، کوئی اور نہیں (0)
$ chmod 600 ~/.ssh/id_rsa

# file کا مالک بدلیں
$ chown alice:developers project.txt

آسان وضاحت

فیکٹری کی Assembly Line

Linux کا ایک سب سے طاقتور خیال یہ ہے کہ tools کو آپس میں جوڑ سکتے ہیں۔ سوچیں ایک فیکٹری: پہلا کارکن کھلونا بناتا ہے، دوسرا رنگ کرتا ہے، تیسرا ڈبے میں رکھتا ہے۔ Pipe | وہ conveyor belt ہے۔ ہر command کا output اگلی command کا input بن جاتا ہے — آسان tools سے طاقتور chains۔

🚰 تین Standard Streams

⌨️

stdin (0)

Standard Input

وہ نل جہاں سے اجزاء آتے ہیں۔ عام طور پر keyboard — لیکن file یا کسی اور command کے output سے بھی آ سکتا ہے۔

🖥️

stdout (1)

Standard Output

وہ نل جہاں سے تیار چیز نکلتی ہے۔ عام طور پر screen — لیکن file میں بھی بھیجی جا سکتی ہے یا اگلی command کو دی جا سکتی ہے۔

🚨

stderr (2)

Standard Error

ایک الگ "اوپس!" نل۔ غلطیوں کے پیغام یہاں آتے ہیں تاکہ انہیں کامیاب output سے الگ محفوظ کیا جا سکے۔

🔗 Connector Symbols

|

Pipe — Conveyor Belt

ایک command کا output براہ راست دوسری command کے input میں ڈالتا ہے۔ Commands کو طاقتور chains میں جوڑتا ہے۔
ls | grep "homework" — files کی فہرست بنائیں، پھر صرف "homework" والی نکالیں۔

>

Output Redirect — پرانا مٹا کر لکھنا

Output کو screen کی بجائے file میں ڈالتا ہے۔ پہلے سے موجود مواد مٹا کر لکھتا ہے۔ جیسے notepad کو مکمل صاف کر کے نیا لکھنا۔
ls -la > file_list.txt

>>

Append Redirect — آخر میں جوڑنا

Output کو file کے آخر میں جوڑتا ہے بغیر پرانا مٹائے۔ جیسے notepad میں اگلا خالی صفحہ پلٹنا۔
echo "new entry" >> logfile.txt

2>

Error Redirect — غلطیاں پکڑیں

صرف "اوپس!" (stderr) پیغامات الگ file میں محفوظ کرتا ہے۔ Output صاف رکھتا ہے جبکہ debugging کے لیے errors محفوظ کرتا ہے۔
./script.sh 2> errors.log

pipe chains — real-world examples
# log میں "error" کتنی بار آیا گنیں
$ cat server.log | grep -i "error" | wc -l
47

# موجودہ folder کی 5 سب سے بڑی files
$ ls -lS | head -n 5

# access log میں unique IP addresses نکالیں
$ cat access.log | awk '{print $1}' | sort | uniq

محفوظ محلے کی دکان

پرانے طریقے میں آپ کسی بھی website سے software download کرتے اور امید رکھتے کہ virus نہ ہو۔ Linux ایک Package Manager استعمال کرتا ہے — آپ کے محلے کی ایک بڑی، محفوظ، جانچی پرکھی دکان۔ ہر چیز safety کے لیے چیک ہوئی ہے۔ Delivery truck (package manager) یہ بھی جانتا ہے کہ کون سی چیز کو کن دوسری چیزوں کی ضرورت ہے اور خود ہی سب لے آتا ہے۔

🍦 Distributions — Linux کے مختلف ذائقے

چونکہ Linux Open Source ہے، کئی مختلف گروہوں نے اپنے versions بنائے — انہیں distros کہتے ہیں۔ سب Linux ہیں (سب آئس کریم ہیں) لیکن مختلف tools اور communities کے ساتھ (مختلف toppings)۔

Distribution FamilyPackage Managerبہترین استعمال
Debian / Ubuntuaptگھریلو computers، cloud servers، نئے لوگوں کے لیے
Red Hat / Fedora / CentOSdnf یا yumبڑی companies، AWS AMIs، corporate ماحول
Arch Linuxpacmanماہر users جو scratch سے سب بنانا چاہیں
apt — Ubuntu/Debian کا delivery truck
# دکان کی فہرست تازہ کریں (نئے versions چیک کریں)
$ sudo apt update

# package install کریں (apt سب ضروری چیزیں خود لاتا ہے)
$ sudo apt install vlc

# تمام installed packages کو latest version پر لائیں
$ sudo apt upgrade

# package ہٹائیں (config files رکھیں)
$ sudo apt remove vlc

📝 Text Editors — Terminal میں لکھنا

Linux کو professionally استعمال کرنے کے لیے آپ کو terminal میں config files بنانی اور ایڈٹ کرنی آنی چاہئیں۔ دو editors مشہور ہیں: ایک نئے لوگوں کے لیے آسان، ایک legendary۔

📗

نئے لوگوں کے لیے

Nano

دوستانہ notepad۔ screen کے نیچے shortcut keys دکھاتا ہے۔ جو لکھیں وہی نظر آئے — بس type کریں اور screen پر نمودار ہو۔

$ nano myfile.txt
# اپنا مواد لکھیں
Ctrl+O → Save کریں
Ctrl+X → باہر نکلیں
🧙

جادوگر کی چھڑی

Vim

مختلف کاموں کے لیے مختلف modes والا editor۔ سیکھنا مشکل ہے، لیکن ایک بار آ جائے تو بہت تیز۔ Vim دنیا کے ہر Linux server پر موجود ہے۔

$ vim myfile.txt
i → Insert Mode (یہاں لکھیں)
Esc → Normal Mode میں واپس
:wq → Save کریں اور نکلیں
:q! → بغیر Save کے نکلیں
dd → موجودہ line مٹائیں

آسان وضاحت

صبح کا معمول

جب آپ power button دباتے ہیں، computer بس "آن" نہیں ہوتا۔ وہ ایک سخت، باترتیب صبح کا معمول پورا کرتا ہے — ہر system کو صحیح ترتیب سے جگاتا ہے تاکہ اگلا شروع ہو سکے۔ جیسے ایک chef کو پہلے عمارت کا تالہ کھولنا ہوتا ہے، پھر چولہا جلانا، پھر team آ کر کام کر سکتی ہے۔

🌅 Boot کے چار مراحل

1

BIOS — الارم گھڑی

Firmware Level

ایک چھوٹا program جو motherboard کی ایک chip میں ہمیشہ کے لیے لکھا ہوا ہے۔ جب بجلی آتی ہے تو یہ جاگتا ہے اور ایک POST (یعنی Power-On Self-Test) کرتا ہے — چیک کرتا ہے کہ چولہا، fridge اور sink سب موجود اور ٹھیک ہیں۔ پھر کوئی bootable device (hard drive یا USB) ڈھونڈتا ہے۔

2

MBR اور GRUB — لباس کا انتخاب

Bootloader

MBR (یعنی Master Boot Record) hard drive کے شروع کا ایک چھوٹا حصہ ہے۔ اس میں اتنا code ہے کہ GRUB (bootloader) تلاش کر کے چلائے۔ اگر آپ کے computer پر Windows اور Linux دونوں ہیں، تو GRUB آپ کو پوچھے گا: آج کون سا operating system پہننا ہے؟

3

Kernel — Master Chef جاگ اٹھا

OS Core

GRUB، Linux kernel کو RAM میں load کرتا ہے۔ Kernel فوری طور پر تمام hardware کا کنٹرول لے لیتا ہے — memory manager شروع کرتا ہے، device drivers چالو کرتا ہے، اور file systems تیار کرتا ہے۔ یہ وہ لمحہ ہے جب Master Chef apron پہنتا ہے، تمام burners جلاتا ہے اور کام کی جگہ تیار کرتا ہے۔

4

Init اور Services — کاروبار کھلا

Userspace

پہلا program جسے kernel چلاتا ہے وہ init ہے (Process ID = 1)۔ جدید systems میں یہ systemd ہوتا ہے۔ Init وہ manager ہے جو kitchen کھولتا ہے — تمام background services صحیح ترتیب سے شروع کرتا ہے: networking، login screen، web server، database۔ جب init کا کام پورا ہو جائے تو آپ کا system استعمال کے لیے تیار ہے۔

$ systemctl status nginx ← service کی حالت دیکھیں
$ systemctl start nginx ← service شروع کریں
$ systemctl enable nginx ← boot پر خود شروع ہو

خلاصہ

عام User سے Master تک

Linux اور Bash انسانی تعاون کی ایک عظیم کامیابی ہے — ایک operating system جو دنیا چلاتا ہے، کروڑوں لوگوں نے مل کر کھلے طریقے سے بنایا، اور سب کے لیے مفت دستیاب ہے۔ تمام layers سمجھ لیں تو آپ passive user سے active master بن جاتے ہیں — کسی بھی machine کے۔

→Architecture: Kernel بطور Master Chef، Bash بطور Waiter
→File System: / سے شروع ایک درخت، ہر کمرے کا مقصد
→Navigation: ls · cd · pwd — دیکھیں اور آگے بڑھیں
→Permissions: تین گروہوں کے لیے تین چابیاں — مکمل کنٹرول
→Pipes: | سے آسان tools کو طاقتور chains میں جوڑیں
→Boot Process: BIOS → GRUB → Kernel → Init — صبح کا معمول

Recipes اب سب کے لیے کھلی ہیں۔ Tools کو لا تعداد طریقوں سے جوڑا جا سکتا ہے۔ Kitchen آپ کا ہے — پکانا شروع کریں۔
